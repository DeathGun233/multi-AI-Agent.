# 第9步：成本看板、批量实验与反馈闭环

## 本次升级目标

这一轮继续强化 AI 应用工程能力，重点不是传统后端能力，而是把项目补到更像企业里的 AI 实验平台：

- 记录模型成本并提供预算看板
- 支持 Prompt / 模型 A/B 批量实验
- 把人工审核反馈回流成评测样本

## 1. 模型成本统计与预算看板

新增能力：

- 每次 LLM 调用记录 `estimated_cost_usd`
- 按月聚合 `total_cost_usd / total_tokens / avg_latency_ms`
- 展示 `monthly_budget_usd`
- 计算 `spend_ratio_percent`
- 计算 `projected_month_end_cost_usd`
- 按模型聚合成本与请求量

页面与接口：

- 页面：`/costs`
- 接口：`GET /api/costs/summary`

核心设计点：

- 成本按 token 单价估算，不依赖外部计费接口
- 预算看板直接复用运行日志里的 LLM 调用明细
- 支持后续扩展到按 Prompt、按工作流、按团队分摊成本

## 2. Prompt A/B 实验批量运行

新增能力：

- 支持一次提交多个变体
- 每个变体可以指定：
  - `model_name`
  - `prompt_profile_id`
  - `routing_policy_id`
- 支持 `repeats`
- 聚合输出：
  - `avg_score`
  - `avg_cost_usd`
  - `avg_latency_ms`
  - `avg_tokens`
  - `handoff_rate`

页面与接口：

- 页面：`/batches`
- 接口：
  - `GET /api/batches`
  - `GET /api/batches/{id}`
  - `POST /api/batches`

核心设计点：

- 批量实验运行不污染正式工作流数据面板，主要用于实验对比
- 每个变体输出保留单次 run 结果，便于后续追溯
- 适合继续扩展为自动 nightly regression

## 3. 人工反馈回流成评测样本

新增能力：

- 人工审核通过或驳回后，自动沉淀 `FeedbackSample`
- 样本保存：
  - 来源运行 ID
  - 工作流类型
  - 输入 payload
  - 人工期望状态
  - 审核人
  - 审核备注
  - 审核分数
  - 期望关键词
  - 输出快照

接口：

- `GET /api/feedback-samples`

评测侧扩展：

- 当存在反馈样本时，自动出现数据集 `feedback-loop-v1`
- 可以直接拿人工反馈样本继续跑自动评测

这让项目形成了一个完整闭环：

1. 工作流运行
2. 人工审核
3. 反馈沉淀
4. 评测数据集扩充
5. 再次对比 Prompt / 模型方案
